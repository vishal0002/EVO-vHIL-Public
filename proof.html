Expected behavior — the watchdog fires on the first silence window before the middleware bridge finishes establishing. Confirms the deadman switch is active immediately from boot, not after a grace period.

Daemon correctly detects SIMULATION build flag and widens the CAN timeout threshold to accommodate virtual bridge latency. Production timeout values are enforced on physical hardware builds.

THERMAL_SHUTDOWN Live

Injected mid-session via middleware. Confirms P1 thermal fault priority fires correctly and propagates through to the daemon's SYS_KILL handler — matching the defined fault priority hierarchy.

Non-Permanent-Latching Verified

The transition from STATE:5 / THERMAL_SHUTDOWN to STATE:4 / SYSTEM_OK between sessions confirms the safety model correctly returns to normal operation once the injected fault clears — without requiring a full cold restart. Frame Format

Extended CAN (29-bit arbitration ID). Each 8-byte payload encodes a structured telemetry snapshot per the proprietary DBC signal map — cell voltages, temperatures, state flags, LV rail data.

Broadcast Cadence

Multiple frames within the same timestamp confirm the BMS firmware's deterministic clock-driven broadcast cadence is being faithfully relayed. Higher-criticality signals transmit at faster intervals.

Zero Packet Loss

Every frame emitted by the BMS USART was received, encoded, and injected across the OS boundary. No drops observed under nominal operating conditions across the 58-minute session.
